# 종합 프로스펙트 분석 (Prospect Analysis Orchestrator)

한국 중소기업 B2B SaaS 영업을 위한 플래그십 분석 스킬입니다. 5개의 서브에이전트를 병렬로 실행하여, 프로스펙트에 대한 360도 종합 분석을 수행합니다.

## 사용법

```
/sales prospect <기업 URL 또는 이름>
/영업 분석 <기업 URL 또는 이름>
```

## 출력 파일

`PROSPECT-ANALYSIS.md`

---

## 실행 아키텍처

### 병렬 에이전트 구성

```
┌─────────────────────────────────────────┐
│        sales-prospect (오케스트레이터)      │
│                                         │
│  1. 디스커버리 단계 (Discovery Phase)      │
│     └─ 웹사이트 분석, 기업 분류, 규모 추정   │
│                                         │
│  2. 병렬 에이전트 실행                      │
│     ├─ sales-company    (기업 분석)       │
│     ├─ sales-contacts   (연락처/의사결정자) │
│     ├─ sales-opportunity (기회 평가)      │
│     ├─ sales-competitive (경쟁 분석)      │
│     └─ sales-strategy   (전략 수립)       │
│                                         │
│  3. 통합 스코어링 및 최종 보고서             │
└─────────────────────────────────────────┘
```

## 1단계: 디스커버리 (Discovery Phase)

### 1-1. 웹사이트 분석

WebFetch로 대상 기업의 웹사이트를 조회하여 아래 정보를 수집하십시오.

| 수집 항목 | 확인 위치 | 비고 |
|-----------|----------|------|
| 기업명 (국/영문) | 메인페이지, 회사소개 | |
| 사업 설명 | 회사소개, About | |
| 주요 제품/서비스 | 제품 페이지, 서비스 페이지 | |
| 팀 규모 신호 | 채용 페이지, 팀 소개 | |
| 고객사 목록 | 고객사 로고, 사례 페이지 | |
| 기술스택 신호 | 소스코드, 채용공고 | |
| 최근 뉴스/블로그 | 뉴스룸, 블로그 | |
| 연락처/위치 | 하단 푸터, 문의 페이지 | |

### 1-2. 기업 분류 (Company Classification)

웹사이트 분석 결과를 기반으로 기업을 분류하십시오.

| 분류 | 정의 | 판별 기준 |
|------|------|-----------|
| SaaS | 클라우드 기반 소프트웨어 제공 | 구독 모델, 무료 체험, 클라우드 언급 |
| IT서비스 | SI/솔루션/컨설팅 | 프로젝트 기반, 수주/납품 언급, 고객사 프로젝트 |
| 제조 | 제품 생산/제조 | 공장, 생산라인, 제품 카탈로그, 인증 |
| 유통/커머스 | 유통/판매/이커머스 | 상품 판매, 쇼핑몰, 물류/배송 |
| 기타 | 위에 해당하지 않는 경우 | 교육, 컨설팅, 미디어, 헬스케어 등 |

### 1-3. 규모 추정 (Size Estimation)

아래 신호를 종합하여 기업 규모를 추정하십시오.

| 신호 | 소규모 (1-20인) | 중소 (20-100인) | 중견 (100-300인) | 중대형 (300인+) |
|------|----------------|----------------|-----------------|----------------|
| 팀 소개 페이지 | 5명 이하 | 부서별 소개 | 조직도 | 계열사 구조 |
| 채용공고 수 | 0-2건 | 3-10건 | 10-30건 | 30건 이상 |
| 사무실 수 | 1곳 | 1-2곳 | 2-5곳 | 5곳 이상 |
| 고객사 수 | 표시 없음 | 10-50개 | 50-200개 | 200개 이상 |
| 매출 추정 | ~10억 | 10-100억 | 100-500억 | 500억 이상 |

## 2단계: 병렬 에이전트 실행 (Parallel Agent Execution)

5개 에이전트를 **동시에** 실행하십시오. 각 에이전트는 독립적으로 작동합니다.

### 에이전트 1: sales-company (기업 분석)
- **입력**: 기업 URL, 디스커버리 결과
- **수행**: 펌그래픽(Firmographic) 분석, 재무 정보 수집, 성장 신호 탐지
- **참조**: `agents/sales-company.md`
- **출력**: 기업 프로필, 재무 건전성, 성장 점수

### 에이전트 2: sales-contacts (의사결정자 매핑)
- **입력**: 기업 URL, 디스커버리 결과
- **수행**: 의사결정 구조 파악, 핵심 인물 식별, 연락처 수집
- **참조**: `agents/sales-contacts.md`
- **출력**: 의사결정자 맵, 연락처, 개인화 앵커

### 에이전트 3: sales-opportunity (기회 평가)
- **입력**: 기업 URL, 디스커버리 결과
- **수행**: BANT + MEDDIC 평가, 구매 신호 탐지, 타이밍 분석
- **참조**: `agents/sales-opportunity.md`
- **출력**: BANT/MEDDIC 스코어, 기회 등급

### 에이전트 4: sales-competitive (경쟁 분석)
- **입력**: 기업 URL, 디스커버리 결과
- **수행**: 현재 솔루션 탐지, 경쟁사 포지셔닝, 전환 가능성 평가
- **참조**: `agents/sales-competitive.md`
- **출력**: 경쟁 환경, 전환 난이도, 포지셔닝 권고

### 에이전트 5: sales-strategy (전략 수립)
- **입력**: 기업 URL, 디스커버리 결과
- **수행**: 최적 접근 채널 선정, 메시지 전략, 아웃리치 시퀀스 설계
- **참조**: `agents/sales-strategy.md`
- **출력**: 아웃리치 전략, 채널별 메시지 초안

## 3단계: 복합 스코어링 (Composite Scoring)

### 가중치 (한국 SMB 최적화)

| 카테고리 | 에이전트 | 가중치 | 근거 |
|---------|---------|--------|------|
| 기업 적합도 (Company Fit) | sales-company | **25%** | 기업 규모, 업종, 성장성이 기본 |
| 연락처 접근성 (Contact Access) | sales-contacts | **15%** | 의사결정자 도달 가능성 |
| 기회 품질 (Opportunity Quality) | sales-opportunity | **20%** | BANT/MEDDIC 적격 여부 |
| 경쟁 포지션 (Competitive Position) | sales-competitive | **15%** | 전환 난이도, 경쟁 우위 |
| 전략 준비도 (Strategy Readiness) | sales-strategy | **25%** | 실행 가능한 접근 전략 존재 여부 |

### 점수 산출 공식

```
종합 점수 = (기업 적합도 × 0.25) + (연락처 접근성 × 0.15) +
           (기회 품질 × 0.20) + (경쟁 포지션 × 0.15) +
           (전략 준비도 × 0.25) + 한국 SMB 가산점
```

### 한국 SMB 가산점 (Korean SMB Bonus Signals)

| 신호 | 가산점 | 조건 |
|------|--------|------|
| 벤처기업 인증 | +3점 | 유효한 벤처인증 보유 |
| 이노비즈/메인비즈 인증 | +2점 | 유효한 인증 보유 |
| 정부지원사업 수혜 | +3점 | 클라우드바우처, DX바우처 등 |
| 최근 투자 유치 | +3점 | 6개월 이내 투자 라운드 |
| 대규모 채용 중 | +2점 | 10건 이상 채용공고 |
| 수출 기업 | +2점 | 수출 매출 비중 30% 이상 |
| 소개/추천 경로 있음 | +5점 | 기존 고객/네트워크 소개 |
| 대표이사 직접 접근 가능 | +3점 | 전화번호/이메일 확보 |
| IT 담당자 있음 | +2점 | CTO/IT팀장 확인 |

> 가산점 합계는 최대 +15점으로 제한합니다. 총점은 100점을 초과할 수 없습니다.

### 등급 기준

| 등급 | 점수 | 의미 | 권장 액션 |
|------|------|------|-----------|
| S | 90-100 | 최우선 프로스펙트 | 즉시 아웃리치, 전담 배정, 1주 내 미팅 |
| A | 80-89 | 우수 프로스펙트 | 1주 내 아웃리치, 개인화 메시지 |
| B+ | 70-79 | 양호 프로스펙트 | 2주 내 아웃리치 |
| B | 60-69 | 보통 프로스펙트 | 너처링(Nurturing) 후 재평가 |
| C | 40-59 | 낮은 우선순위 | 마케팅 너처링만 |
| D | 0-39 | 부적합 | 보류 또는 제외 |

## 에이전트 간 데이터 공유 규칙

1. 디스커버리 단계의 결과는 모든 에이전트에 공유됩니다.
2. 각 에이전트는 독립적으로 실행되므로, 다른 에이전트의 결과에 의존하지 마십시오.
3. 최종 스코어링은 오케스트레이터(본 스킬)가 5개 에이전트 결과를 수집한 후 수행합니다.
4. 에이전트 실행 중 오류가 발생하면, 해당 카테고리는 "정보 부족"으로 표시하고 나머지 결과로 점수를 산출합니다.

|------|-----------|

## 주의사항

- 모든 정보는 공개 소스에서 수집합니다.
- 추정치는 반드시 "추정"으로 표시하십시오.
- 발송 준비 메시지는 실제 발송 전에 반드시 사용자의 검토를 받으십시오.
- 점수가 낮더라도 단기적 매출 기회가 아닌 장기적 관계 구축 관점에서 평가하십시오.
