# URL Builder

## 목적

정식 구현 템플릿이 되기 전, 사전 검토를 위해 HTML로 렌더링한 결과물을 프로젝트가 실제로 제공하는 URL로 바꾼다. 결과물의 대상은 기능 계획, 화면 흐름, 정책 검토, 데이터 구조, 구현 기준 등 무엇이든 될 수 있다. 파일 경로를 `localhost:8000` 뒤에 그대로 붙이지 말고, 기존 URL 패턴이나 공통 URL 생성 함수를 사용한다.

일반 Django 화면 이동 URL, API URL, 운영 기능 링크를 찾는 용도가 아니다. 그런 경우에는 해당 앱의 URL 패턴이나 `reverse()`를 직접 확인한다.

## 절차

1. 사용자가 파일, 경로, 문서, 라우트, 화면 이름을 지정하지 않고 "URL 만들어줘"처럼 일반 요청만 하면 `http://localhost:8000/`을 답한다.
2. 현재 작업 폴더와 프로젝트 지침에서 `<프로젝트이름>`을 확인한다.
3. `AGENTS.md`, `CLAUDE.md`, `README`, 라우팅 파일에서 사전 검토용 HTML URL 규칙을 찾는다.
4. 요청 파일이 정식 구현 템플릿이 아니라 사전 검토용 HTML 결과물인지 확인한다.
5. 프로젝트 지침에 URL 생성 함수가 있으면 그 함수를 사용한다.
6. 프로젝트 지침에 고정 URL 패턴이 있으면 그 패턴에 파일의 프로젝트 내부 상대 경로를 넣는다.
7. 규칙이 없으면 URL을 만들지 말고, 해당 파일에 대한 브라우저 URL 규칙이 없다고 말한다.

## 공통 규칙

- URL만 만든다. 서버 상태 확인, 서버 실행, 서버 중지는 하지 않는다.
- 파일, 경로, 문서, 라우트, 화면 이름이 없는 일반 URL 요청의 기본값은 `http://localhost:8000/`이다.
- `file://` URL을 주지 않는다.
- 사전 검토용 HTML로 제공되는 기존 URL 패턴이 없으면 URL을 임의 생성하지 말고, 해당 파일은 아직 URL 대상이 아니라고 말한다.
- 파일이 실제로 존재하는지 확인하는 것은 가능하지만, 서버 포트 확인은 하지 않는다.

## 프로젝트 지침에서 찾을 것

프로젝트마다 아래 같은 매핑이 있을 수 있다. `<...>` 값은 프로젝트 지침에서 가져온다.

| 지침 항목 | 의미 |
| --- | --- |
| `<검토HTML파일패턴>` | 사전 검토용 HTML 결과물이 놓이는 위치 |
| `<URL생성함수>` | 사전 검토용 HTML URL을 계산하는 공통 함수 |
| `<URL접두어>` | 사전 검토용 HTML을 여는 고정 URL 앞부분 |
| `<기본호스트>` | 사용자가 개발 서버를 켰을 때 접속하는 호스트 |

예시는 형태만 보여준다. 실제 값은 현재 프로젝트 지침을 따른다.

```text
<프로젝트이름>의 <검토HTML파일패턴>/<문서>.html
→ <기본호스트>/<URL접두어>/<검토HTML파일패턴>/<문서>.html
```

한글·공백 경로는 직접 인코딩하지 말고 프로젝트의 `<URL생성함수>`나 표준 URL 유틸을 사용한다.

## 답변 형식

URL을 먼저 보여주고, 필요하면 한 문장만 덧붙인다.

```text
<브라우저 URL>

개발 서버가 켜져 있어야 브라우저에서 열립니다.
```
