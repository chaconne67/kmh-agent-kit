# 60초 빠른 평가 (Quick Assessment)

서브에이전트를 활성화하지 않고, 최소한의 조사만으로 프로스펙트의 적합도를 60초 내에 빠르게 평가하는 스킬입니다.

## 사용법

```
/sales quick <기업 URL 또는 이름>
/영업 빠른분석 <기업 URL 또는 이름>
```

## 출력

콘솔 출력 (500단어 이내) — 별도 파일 생성 없음, 서브에이전트 활성화 없음

---

## 실행 절차

### 1단계: 빠른 정보 수집 (30초)

아래 2가지만 수행하십시오. 추가 조사는 하지 않습니다.

#### 1-1. 웹사이트 분석
- WebFetch로 대상 기업 웹사이트 메인페이지를 1회 조회합니다.
- 수집 항목: 기업명, 사업 설명, 주요 제품/서비스, 규모 신호, 기술 수준 신호

#### 1-2. 기본 검색
- WebSearch로 `"[기업명]" 기업 정보` 1회 검색합니다.
- 수집 항목: 업종, 설립연도, 직원 수, 최근 뉴스, 투자 이력

### 2단계: 즉시 신호 판별 (15초)

수집된 정보에서 아래 신호를 빠르게 체크하십시오.

#### 긍정 신호 (Green Flags)
- [ ] 우리 솔루션의 타겟 업종에 해당
- [ ] 적절한 규모 (너무 작지도, 너무 크지도 않음)
- [ ] 성장 중인 기업 (채용, 투자, 신사업)
- [ ] 기술 친화적 (SaaS 사용, 클라우드 활용)
- [ ] 정부 인증/지원사업 수혜 신호
- [ ] 의사결정자 접근 가능성
- [ ] 명확한 고통점 존재 가능성

#### 부정 신호 (Red Flags)
- [ ] 타겟 업종 외
- [ ] 규모 부적합 (너무 소규모 또는 대기업)
- [ ] 정체/축소 신호 (구조조정, 매출 감소)
- [ ] 기술 수준 극히 낮음
- [ ] 경쟁사와 장기 계약 중 (확인 가능한 경우)
- [ ] 접근 경로 전무

### 3단계: 점수 추정 및 출력 (15초)

수집된 신호를 기반으로 점수를 추정하고, 아래 형식으로 콘솔에 출력하십시오.

---

## 출력 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚡ 빠른 평가: [기업명]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

점수: [XX]/100 ([등급]) | [기업 유형] | [추정 규모]

■ 강점 Top 3
  1. [강점 1 — 한 줄 설명]
  2. [강점 2 — 한 줄 설명]
  3. [강점 3 — 한 줄 설명]

■ 우려 Top 3
  1. [우려 1 — 한 줄 설명]
  2. [우려 2 — 한 줄 설명]
  3. [우려 3 — 한 줄 설명]

■ 추천 다음 단계
  → [구체적인 한 가지 권장 액션]

■ 심층 분석 필요 여부
  → [예/아니오] — [이유]
  → 심층 분석 명령어: /sales prospect [URL]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 등급 기준 (빠른 평가용)

| 등급 | 점수 | 의미 |
|------|------|------|
| S | 85-100 | 즉시 심층 분석 권장 |
| A | 70-84 | 심층 분석 권장 |
| B | 55-69 | 추가 정보 확인 후 판단 |
| C | 40-54 | 우선순위 낮음 |
| D | 0-39 | 부적합 — 보류 권장 |

## 주의사항

- **서브에이전트를 실행하지 마십시오.** 이 스킬의 핵심은 속도입니다.
- WebFetch 1회, WebSearch 1회로 제한하십시오.
- 출력은 500단어 이내로 유지하십시오.
- 정보가 부족한 경우 "확인 필요"로 표시하고, 심층 분석(`/sales prospect`)을 권장하십시오.
- 점수는 제한된 정보 기반의 **추정치**임을 명시하십시오.
- 빠른 판단이 목적이므로, 완벽한 분석보다 **방향성 판단**에 집중하십시오.
