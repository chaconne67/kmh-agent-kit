# 파이프라인 요약 리포트 (Sales Pipeline Report)

현재 디렉토리에 있는 분석 결과 파일들을 스캔하고 종합하여, 전체 파이프라인 현황을 한눈에 파악할 수 있는 세일즈 리포트를 생성하는 스킬입니다.

## 사용법

```
/sales report
/영업 리포트
```

## 출력 파일

`SALES-REPORT.md`

---

## 실행 절차

### 1단계: 분석 파일 스캔 (File Scan)

현재 작업 디렉토리 및 하위 디렉토리에서 아래 파일들을 탐색하십시오.

| 파일명 | 생성 스킬 | 내용 |
|--------|----------|------|
| `PROSPECT-ANALYSIS.md` | sales-prospect | 종합 프로스펙트 분석 |
| `COMPANY-RESEARCH.md` | sales-research | 기업 조사 |
| `LEAD-QUALIFICATION.md` | sales-qualify | 리드 자격 판정 |
| `DECISION-MAKERS.md` | sales-contacts | 의사결정자 매핑 |
| `OUTREACH-SEQUENCE.md` | sales-outreach | 아웃리치 시퀀스 |
| `FOLLOWUP-SEQUENCE.md` | sales-followup | 후속 연락 시퀀스 |
| `MEETING-PREP.md` | sales-prep | 미팅 준비 브리프 |
| `CLIENT-PROPOSAL.md` | sales-proposal | 제안서 |
| `COMPETITIVE-INTEL.md` | sales-competitors | 경쟁사 분석 |
| `IDEAL-CUSTOMER-PROFILE.md` | sales-icp | ICP |

각 파일에서 핵심 데이터(기업명, 점수, 등급, 상태, 다음 단계 등)를 추출하십시오.

### 2단계: 리포트 생성 (Report Generation)

추출된 데이터를 종합하여 `SALES-REPORT.md` 파일을 아래 8개 섹션으로 작성하십시오.

---

먼저 프로스펙트 분석을 실행해 주세요:
  /sales prospect <기업 URL>
  /sales quick <기업 URL>

분석 파일이 생성된 후 다시 리포트를 실행하시면
파이프라인 현황을 종합해 드리겠습니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 주의사항

- 리포트는 기존 분석 파일의 데이터만 활용합니다. 새로운 조사를 수행하지 않습니다.
- 파일이 일부만 존재하는 경우, 존재하는 파일 기반으로 가능한 범위 내에서 리포트를 작성하십시오.
- 데이터가 없는 섹션은 "데이터 없음"으로 표시하고, 해당 분석을 실행할 수 있는 명령어를 안내하십시오.
- 리포트 생성 시 기존 `SALES-REPORT.md` 파일이 있으면 덮어씁니다.
