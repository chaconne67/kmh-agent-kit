# 코드 리뷰 루프

## 역할

`$code-review`를 반복 실행하는 얇은 래퍼다.

- 리뷰 계약, 후보 의혹, finding 승인, 열린 질문, 리뷰 출력은 `$code-review`를 그대로 따른다.
- 이 스킬에서 별도의 리뷰 기준, finding 원장, 후보 분류를 만들지 않는다.
- 이 루프를 맡은 메인 에이전트가 `$code-review` 실행, 수정, 검증을 직접 수행한다.
- 서브에이전트, 리프 리뷰어, 다른 리뷰 경로를 만들지 않는다.

## 반복 중 유지할 값

최초 리뷰 전에 다음 값을 기록한다.

1. 사용자 목적과 승인된 기준 원천
2. 기준 base와 리뷰 대상 PR 또는 로컬 diff
3. 허용된 리뷰·수정 범위
4. 적용 지침과 필수 검증

모든 라운드에 같은 사용자 목적, 기준 base, 승인된 범위를 전달한다.
수정으로 바뀐 head 또는 작업 트리는 다음 라운드의 갱신된 diff로 사용한다.

## 실행 루프

1. 현재 PR 또는 로컬 diff를 대상으로 `$code-review`를 실행한다.
2. 반환된 승인 finding을 비판적으로 검토해서 타당한 finding만 수정한다.
3. `material contract question`이나 계약 변경 항목은 수정하지 않고 열린 질문으로 남긴다.
4. 수정한 내용이 있으면 같은 목적과 범위로 갱신된 diff 전체를 대상으로 1단계부터 반복한다.
5. 승인 finding이 없으면 완료 조건을 확인한다.

각 재리뷰는 `$code-review`의 새 단일 리뷰 패스다.
이전 라운드의 finding만 확인하지 말고 갱신된 diff에서 새로 승인되는 finding도 처리한다.

## 중단 조건

다음 중 하나가 발생하면 성공으로 처리하지 말고 중단 사유를 보고한다.

- 리뷰할 diff가 없다.
- 사용자 답이 필요한 열린 질문이 있다.
- finding을 고치려면 승인된 목적이나 수정 범위를 바꿔야 한다.
- 필수 검증이 실패했고 허용된 범위 안의 안전한 수정이 명확하지 않다.
- 같은 finding이 진전 없이 반복된다.

## 완료 조건

다음 조건을 모두 만족하면 루프를 끝낸다.

- 마지막 `$code-review`에서 승인된 finding이 없다.
- 열린 `material contract question`과 계약 변경 항목이 없다.
- 수정에 필요한 검증이 모두 통과했다.
- 적용 지침의 완료 조건을 충족했다.

최종 보고에는 수정한 finding, 검증 결과, 마지막 코드 리뷰 결과를 남긴다.
